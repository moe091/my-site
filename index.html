A Bit About Me

I tried to keep this short, but have evidently failed. I urge you to at least skim through it anyway - it's not so long as it looks!

I'm basically just a normal guy, I love learning new things, creating things, and programming. I'm pretty good at problem-solving, pretty bad at art(but getting better - I'm practicing art so I can design and create better software), great at communicating and getting along or working with others. Right now my main goal, aside from completing my current software projects, is to join a team and work as a professional developer. I want to interact with other developers, learn from them, and make something awesome as part of a team; I have been preparing for such an opportunity and am pretty confident that I could be a very valuable and productive member of a development team. Here's a few thoughts about my experience with programming and software development: I first got into programming because I loved technology and games, and I thought coding was 'cool'(or maybe I thought it was 'Da Bomb,' I don't really remember and this was in the 90s so it's very possible) I kept writing code because I was good at it; logic and problem solving were/are my strengths and, being the show-offy youngster I was, I loved doing stuff I was good at programming is still my passion because I still love technology and games, I'm still good at logic and problem solving(but fortunately not such a showoff anymore; life has certainly humbled me), and because as an adult I found that I love creating things and that's what I want to do with my life - programming is an extremely creative tool with infinite possibilities for creation, it's awesome My desire to create things lead me to broaden and improve my programming skills. I wanted to create mobile apps, games, websites, AI algorithms, etc, so I learned how to. experiencing many different types of software development helped me understand how to create better software and write better code in general My journey in software dev has lead me to develop many other adjacent skills. I got good better at visual art by creating graphics for games and designing websites, I learned how to think about and apply principles of UI and UX design so I could make better websites and apps, and I learned how to engineer fun in my many attempts at game design(a skill which I am currently applying in a different way - in order to improve the user experience in my current web app project and motivate users to use my app to improve their eating and nutrition habits) This journey(plus a lot of hard work) has lead me to become a skilled creator of websites, apps, and games. I know how to write good code, but I also know how to design a good user experience, create aesthetics that compliment the software's functionality, and communicate with others to work together and convert an idea into quality software
