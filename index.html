Merkle Brown - Javascript HTML5 Game

Merkle Brown

Tech:
Javascript - Phaser - Canvas - WebGL - GLSL - JQuery

Overview

Merkle Brown is my current 'secondary' side-project, that I work on when I need a break from my main project. I plan on releasing it on several platforms as a commercial indie game when it's complete, using different wrappers to deploy it on Steam/Windows, Android, iOS, and of course I'll release it as a browser-based html5 game as well - it will even run in mobile web browsers without any downloads or plugins.

Features

Merkle Brown is a 2D, physics-based platformer game with realtime physics simulation, custom hardware-accelerated shaders for lighting and other effects, tons of complicated dynamic and scripted game objects, and an extremely capable(albeit ugly) level editor that allows you to build dynamic obstacles by combining different bodies, triggers, effects, actions, movepaths, joints, springs, and more, allowing endless possibilities for different creative and fun game objects. It's also possible playtest levels as you create them inside the editor, allowing for quick iteration and testing - which is crucial when designing non-tile based levels for a platformer game and you need to position things just right in order to perfect the difficulty curve.

Progress

Although the engine itself is mostly complete and editor and other game code is pretty far along, none of the levels or graphics I have created at this point will end up in the final game. I've written all the tools I need to create an awesome and fun game, and now I just need to put it all together into a cohesive whole using quality assets and exceptional level & graphic design. My art skills were HORRIBLE when I started this project, and although they've come a very long way, I don't think I'm capable of doing the game justice with my art yet. I'm much better with level design, and have made some truly interesting and enjoyable levels already while building and testing the different features of my engine and editor, but I really want to focus on getting the level design absolutely perfect as I feel that is the single most crucial factor in making a quality platformer game, and I'm not quite there yet. I plan on continuing to do what I've done this far, which is to keep iterating on the graphics and level design as a add features and improve my engine and editor, pushing myself to learn more and improve a lot on each iteration, until I finally feel like I'm capable of creating a truly top-quality game that stands out in every category. I've come such a long way from when I first started, and this project has truly been an emotional journey, but I know if I keep improving as I have been and stay true to my vision I will end up with an amazing game that I can be extremely proud of
NuTrack - Web App

NuTrack

Tech:
Javascript - ES6 - Node - Express - React - React Router - Bootstrap - SASS - HTML5 - CSS3 - Passport - Mocha - Enzyme - Should - Webpack - Babel - USDA Nutrient API

Overview

NuTrack is a Single-Page Web Application made with Node, MongoDB, Express, and React. It retrieves nutrient data on the backend from the USDA Nutrient Data API and serves it with it's own custom API that adds many more features. The purpose of the app is to provide a flexible and easy-to-use tool that users can use to plan their food/nutrient consumption and track their progress. When I first started the project my plan was to make something with the USDA API mostly just to practice and learn some new technologies, but before long I ended up finding my own uses for the App and coming up with tons of useful new features that I wanted to add, and the project soon become something much larger and more useful than I had originally planned. I use it mainly to make sure I am getting enough calories to keep my weight up but it also provides other insights about my nutrition patterns automatically and has made me more aware of how my eating habits are effecting my health.

NuTrack is basically in the "Minimum-Viable-Product" phase right now, it works as a cohesive application and has many useful features, but the more I work on it the bigger my vision for the project becomes - I now have a pretty long todo-list of tweaks and new features I want to implement, some of which will improve the usability and capabilities of the app drastically. Once I implement a few of the larger, more impactful features that I have planned, I intend to set up real hosting for the app and release to the public as a commercial web app with trial accounts available for free and premium accounts available for a small monthly subscription fee. Right now I don't think NuTrack is useful enough for many people to want to pay for it; the limits of the USDA Database(mainly the lack of many popular food products) as well the lack of a few key features I plan on implementing and the mediocre, quickly made design of the current site are the only things holding it back from being a high-quality, useful, commercial web application. Luckily, all of these things are fixable - the limitations of the USDA Database will be the most difficult, but I have a fun and interesting solution planned to fix that problem.

About Me

My passion for creating things is what first drew me into programming.
